# Security policy

ReplicaFuzz is an experimental alpha. Please do not include secrets, private
application data, credentials, or proprietary replay artifacts in public issues.

For vulnerabilities in ReplicaFuzz itself, use GitHub's private vulnerability
reporting flow at:

https://github.com/Atomics-hub/replicafuzz/security/advisories/new

The maintainer will acknowledge a report within seven days. Only the latest
alpha release is supported while the project remains pre-1.0.

Failures discovered in an application under test belong to that application's
security process; ReplicaFuzz does not authorize testing systems you do not own
or have permission to assess.
